# MVP Counter Application (wxPython)

Простое настольное приложение-счетчик с системой логирования, спроектированное по классическому архитектурному паттерну **Model-View-Presenter (MVP)** с использованием графической библиотеки **wxPython**.

Проект демонстрирует разделение бизнес-логики и интерфейса, использование паттерна *Наблюдатель (Observer)* для синхронизации нескольких окон, а также изоляцию слоев для удобного модульного тестирования.

## 🛠 Архитектурная структура (MVP)

*   **Model (Модель)**: Чистая бизнес-логика. Хранит состояние счетчика и выполняет операции. Ничего не знает о GUI (`wxPython`).
*   **View (Представление)**: Пассивный графический интерфейс. Отвечает только за отрисовку виджетов и пересылку событий пользователя Презентеру.
*   **Presenter (Презентер)**: Посредник. Реагирует на действия пользователя из View, обновляет Model и синхронизирует отображение на экране.

### Структура каталогов

```text
my_mvp_project/
├── .venv/                   # Локальное виртуальное окружение
├── app/                     # Основной пакет приложения
│   ├── model/              # Слой данных (Model)
│   ├── presenter/          # Слой посредников (Presenter)
│   └── view/               # Графические окна (View)
├── tests/                   # Юнит-тесты (с использованием Mock-объектов)
├── .gitignore               # Исключения для Git (кэш, .venv, настройки IDE)
├── README.md                # Описание проекта
├── requirements.txt         # Список внешних зависимостей
└── run.py                   # Главная точка входа для запуска приложения
```

## 🚀 Быстрый запуск

### 1. Клонирование и подготовка окружения
Перейдите в папку с проектом и создайте виртуальное окружение:

```bash
# Создание окружения
py -m venv .venv

# Активация для Windows:
.venv\Scripts\activate

# Активация для macOS/Linux:
source .venv/bin/activate
```

### 2. Установка зависимостей
Установите библиотеку `wxPython` одной командой из файла зависимостей:

```bash
py -m pip install -r requirements.txt
```

### 3. Запуск приложения
Запустите главный скрипт из корневой директории:

```bash
py run.py
```

При запуске откроются два независимых окна, работающих с общей моделью данных:
1.  **Управление счетчиком**: Окно с кнопками `+`, `-` и `Сброс`.
2.  **История изменений (Лог)**: Окно, отображающее хронику всех действий в реальном времени.

## 🧪 Тестирование

Проект покрыт модульными тестами. Презентеры тестируются в полной изоляции от графической оболочки за счет использования `unittest.mock`.

Для запуска тестов выполните в терминале команду:

```bash
py -m unittest discover -s tests
```
