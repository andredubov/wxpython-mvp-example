# wxPython MVP Example

Демонстрационное приложение на Python с использованием библиотеки **wxPython** и архитектурного паттерна **MVP (Model-View-Presenter)**. Проект создан как пример чистой архитектуры и разделения ответственности в настольных приложениях.

## Идея проекта

Цель проекта — показать, как можно построить настольное приложение с графическим интерфейсом, используя паттерн MVP для чёткого разделения:

- **Model** — бизнес-логика и данные
- **View** — пользовательский интерфейс (wxPython фреймы)
- **Presenter** — связующее звено между View и Model, обрабатывающее пользовательские действия и обновляющее представление

Приложение реализует простой счётчик с возможностью увеличения, уменьшения и сброса значения, а также ведение лога действий.

## Функционал

- ✅ Главное окно с отображением текущего значения счётчика
- ✅ Кнопки управления: **Увеличить**, **Уменьшить**, **Сбросить**
- ✅ Окно журнала событий (лог действий пользователя)
- ✅ Маршрутизация между окнами через встроенный `Router`
- ✅ Конфигурация приложения через JSON-файл
- ✅ Поддержка локализации (русский язык)
- ✅ Кроссплатформенность (Windows / Linux)

## Установка и запуск

### Требования

- Python 3.7 (32-bit) — **строгое требование** для сборки через `build.bat`
- pip (менеджер пакетов)

### Windows

1. Клонируйте репозиторий:

   ```bash
   git clone <url-репозитория>
   cd wxpython-mvp-example
   ```

2. Создайте и активируйте виртуальное окружение (рекомендуется):

   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   ```

   > **Примечание:** Скрипт сборки `build.bat` автоматически создаёт виртуальное окружение `.venv`, поэтому этот шаг можно пропустить, если вы планируете использовать `build.bat`.

3. Установите зависимости (для Windows x86):

   ```bash
   pip install -r requirements/windows/x86/requirements.txt
   ```

   > **Примечание:** Скрипт сборки `build.bat` использует зависимости из `requirements/windows/x86/requirements.txt`. Для ручного запуска приложения без сборки можно использовать `requirements/windows/x64/requirements.txt`, если у вас 64-битная версия Python.

4. Запустите приложение:

   ```bash
   python run.py
   ```

   Либо используйте скрипт сборки (требуется Python 3.7 32-bit):
   ```bash
   build.bat
   ```

   > **Важно:** `build.bat` автоматически создаёт виртуальное окружение `.venv`, устанавливает зависимости и собирает исполняемый файл с помощью Nuitka.

### Linux

1. Клонируйте репозиторий:

   ```bash
   git clone <url-репозитория>
   cd wxpython-mvp-example
   ```

2. Создайте и активируйте виртуальное окружение (рекомендуется):

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```

3. Установите зависимости:

   ```bash
   pip install -r requirements/linux/requirements.txt
   ```

4. Запустите приложение:

   ```bash
   python run.py
   ```

   Либо используйте Makefile:
   ```bash
   make run
   ```

## Особенности, требующие внимания

### 1. Маршрутизация между окнами

Переключение между окнами осуществляется через `AppRouter`. Все переходы должны инициироваться через роутер, чтобы сохранить единый поток управления.

```python
# Пример использования
self.router.navigate_to("main")
```

### 2. Разделение ответственности

- **Модель** содержит только бизнес-логику и данные. Она ничего не знает о пользовательском интерфейсе.
- **Презентер** обрабатывает команды от View, вызывает методы Модели и обновляет View.
- **View** отвечает только за отображение и передачу пользовательских действий Презентеру.

### 3. Локализация

Приложение поддерживает русский язык. Файлы перевода находятся в `app/locales/ru/LC_MESSAGES/app.mo`. Для добавления нового языка необходимо:

1. Создать `.po` файл для языка
2. Скомпилировать его в `.mo` с помощью `msgfmt`
3. Поместить в соответствующую директорию `locales/<lang>/LC_MESSAGES/`

### 4. Конфигурация

Настройки приложения хранятся в `app/assets/config.json`. При изменении конфигурации необходимо перезапустить приложение.

### 5. Тестирование

Тесты находятся в каталоге `tests/`. Для запуска тестов используйте:

```bash
python -m unittest discover tests
```

### 6. Сборка

- Для Windows: используйте `build.bat` (создаёт исполняемый файл с помощью Nuitka)
- Для Linux: используйте `make build` (создаёт исполняемый файл)

#### Подробнее о `build.bat`:

Скрипт сборки для Windows использует **Nuitka** (современный компилятор Python) и имеет следующие особенности:

- **Требует Python 3.7 32-bit** — строгая проверка версии и разрядности
- Создаёт виртуальное окружение `.venv` автоматически
- Устанавливает зависимости из `requirements/windows/x86/requirements.txt`
- Собирает один исполняемый файл (режим `onefile`) без консольного окна
- Включает иконку из `app/assets/icons/app-icon.ico` (если есть)
- Включает splash screen из `app/assets/images/logo.png` (если есть)
- Копирует папку `app/assets` в собранный файл

**Интерактивное меню** `build.bat`:
1. Собрать исполняемый файл
2. Запустить исполняемый файл
3. Очистка проекта (удаляет `.build`, `.venv`, `__pycache__` и временные файлы)
4. Выход
